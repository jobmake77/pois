# Pois Art

Pois Art 是一个移动端优先的照片海报生成 Web 应用。当前版本已经收敛到更准确的 `单图 / 双图编辑器` 模型：先把主画板、照片裁切、可选填充块和波点生成这几件事做准，再继续往更多拼法扩展。

## 当前阶段

当前项目处于 `Alpha / 可运行原型修订版`。

这一轮已经完成的重点是：

- 编辑器收敛成 `一个真实主画板`
- 去掉旧的玻璃蒙版、提示文案、额外覆盖框
- 单图默认进入 `照片 + 填充块 + 左右分块`
- 双图默认进入 `双图`
- 裁切升级为 `x / y / scale`，照片始终按 `cover` 填满所在块
- Web 端支持滚轮缩放 + 拖动，移动端支持单指拖动 + 双指缩放
- 右侧只保留真实可编辑面板：`布局 / 填充块 / 波点`

更详细的实现状态见：

- [当前实现进展文档](./docs/IMPLEMENTATION_STATUS.md)

## 当前已完成内容

### 1. 产品主流程

- 首页只保留 hero + 上传入口
- 上传图片后自动进入编辑页
- 单图自动生成首版，不要求用户先理解参数
- 支持随机、切换布局、开关填充块、切换波点风格
- 支持高清导出、下载、系统分享和复制分享文案

### 2. 编辑器模型

- 当前只支持以下准确模式：
  - `单图`
  - `单图 + 填充块`
  - `双图`
  - `双图 + 填充块`
- 当前 UI 里把旧的"主题块"统一改成 `填充块`
- 单图默认：
  - 启用填充块
  - 默认方向为左右分块
- 双图默认：
  - 不自动叠填充块
  - 可手动打开填充块

### 3. 图片编辑能力

- 顶部图片栏显示当前素材卡片
- 每张卡片右上角可以直接删除
- 当前版本最多支持 `2 张照片`
- 点击图片卡片可切换当前编辑图
- 点击画板中的照片块也可切换当前编辑图
- 当前激活照片只显示轻量边界高亮
- 右上角只保留一个小型缩放比例浮层

### 4. 裁切与交互

- `photoCrops` 已升级为：
  - `x`
  - `y`
  - `scale`
- 渲染使用严格 `cover + clamp`
- 不允许出现边缘漏白
- Web 端：
  - 鼠标滚轮缩放
  - 按住拖动平移
- 移动端：
  - 单指拖动
  - 双指 pinch 缩放

### 5. 波点与填充块

- 填充块是画板中的真实块，不是背景层，也不是蒙版
- 填充块可以：
  - 开启 / 关闭
  - 随布局切换左右 / 上下
  - 单独开关是否显示波点
  - 切换底板样式和颜色
- 波点支持：
  - 五角星 / 水滴 / 雪花 / 圆点 / 方块
  - 图片切片 / 颜色采样 / 纯色填充
  - 随机散布 / grid / bottom-heavy
- 当前默认参数已上调：
  - 点更大
  - 点更多
  - 装饰点更明显

## 当前限制

- 当前先收敛到 `最多 2 张照片`，更复杂的多图拼贴不在此阶段范围内
- 还没有撤销 / 重做
- 还没有自动化测试
- 真机上的细致手势手感还需要继续打磨

## 项目结构

```text
src/
  App.tsx                     应用状态与主流程
  components/
    HomeScreen.tsx            首页
    EditorScreen.tsx          编辑页
    ExportSheet.tsx           导出结果浮层
  render/
    blockLayout.ts            单图 / 双图 / 填充块布局求解
    crop.ts                   照片裁切几何与约束
    engine.ts                 画板渲染主引擎
    random.ts                 随机与数学工具
    shapes.ts                 形状路径生成
    worker.ts                 导出 Worker
    workerClient.ts           Worker 调用封装
  presets.ts                  默认参数与主题模板
  types.ts                    核心类型定义
  styles.css                  全局样式与移动端适配
docs/
  IMPLEMENTATION_STATUS.md    当前阶段说明
```

## 本地运行

```bash
npm install
npm run dev
```

生产构建：

```bash
npm run build
npm run preview
```

## 说明

- 当前版本的重点不是功能数量，而是把基础编辑体验做准。
- 下一阶段建议优先继续打磨真机手感、细化桌面端缩放体验，并补一轮基础自动化验证。
