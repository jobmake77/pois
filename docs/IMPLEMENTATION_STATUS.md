# Pois Art 当前实现进展

## 1. 当前阶段判断

当前项目已经进入 `第一版可运行原型的交互校准期`。

和上一阶段相比，这一轮最大的变化不是继续扩展更多拼图模板，而是主动收敛模型，把编辑器从“上图 + 下底板 + 额外提示层”的混合状态，改成了更准确的：

- `一个主画板`
- `照片块`
- `可选填充块`

现在更准确的定位是：

- `核心玩法已跑通`
- `单图 / 双图编辑模型已落地`
- `裁切与缩放逻辑已重做`
- `接下来重点是交互细节和真机手感打磨`

## 2. 这一轮已经完成的能力

### 首页

- 保留 hero
- 保留上传入口
- 保留 2 张开场示例
- 去掉了旧的 Demo 平铺逻辑

### 编辑页结构

- 桌面端改成 `左侧主画板 + 右侧编辑面板`
- 移除旧的状态型信息卡：
  - 当前布局
  - 当前主题块
  - 最近一次预览
- 右侧只保留真实可编辑内容：
  - `布局`
  - `填充块`
  - `波点`
- 移动端改成：
  - 图片栏
  - 主画板
  - 可切换的折叠面板

### 图片栏

- 删除行为已稳定放回顶部图片栏
- 每张图片卡片右上角都有 `×`
- 点击卡片切换当前编辑图
- 当前版本最多支持 `2 张图片`
- 删除到 0 张时会回到首页

### 画板模型

当前画板只支持以下 4 种准确模式：

- `单图`
- `单图 + 填充块`
- `双图`
- `双图 + 填充块`

默认行为：

- 上传 1 张图：
  - 自动进入 `单图 + 填充块`
  - 默认方向为 `左右分块`
- 上传 2 张图：
  - 自动进入 `双图`
  - 用户可手动打开填充块

## 3. 当前已经完成的技术实现

### 状态模型

已收敛成新的核心状态：

- `layoutMode`
- `layoutDirection`
- `fillBlockEnabled`
- `fillBlockDotsEnabled`
- `activePhotoId`
- `photoCrops[x, y, scale]`

旧的这些泛化字段已经退出当前主模型：

- `blocks`
- `layoutPreset`
- 三拼 / 四拼相关状态语义

### 布局求解

当前布局求解集中在：

- [blockLayout.ts](/Users/a77/Desktop/pois/src/render/blockLayout.ts)

它现在负责：

- 单图是否启用填充块
- 双图是否启用填充块
- 左右 / 上下方向下的块矩形求解

### 渲染引擎

当前渲染逻辑集中在：

- [engine.ts](/Users/a77/Desktop/pois/src/render/engine.ts)

本轮已完成：

- 照片块按严格 `cover` 裁切渲染
- 裁切参数升级为 `x / y / scale`
- 渲染时做 clamp，保证不露白
- 填充块只在自己的区域内绘制底板和波点
- 波点默认值整体上调，更大、更密、更明显

### 编辑交互

当前画板已经完成：

- 点击照片块切换当前编辑图
- Web 端滚轮缩放
- Web 端按住拖动平移
- 移动端单指拖动
- 移动端双指缩放
- 右上角小型缩放比例浮层

当前已明确删除：

- 拖动提示文案
- 玻璃蒙版
- 二次裁切框
- 额外覆盖层

## 4. 当前支持的产品能力

### 上传与素材管理

- 上传即生成
- 最多 2 张图
- 顶部图片栏切换当前图
- 顶部图片栏直接删除

### 布局

- 单图
- 左右分块
- 上下分块
- 双图
- 双图 + 填充块

### 填充块

- 开启 / 关闭
- 随方向切换左右 / 上下
- 可单独开关波点
- 可切换底板样式和颜色

### 波点设置

- 五角星 / 水滴 / 雪花 / 圆点 / 方块
- 大小
- 大小差异
- 总点数
- 装饰点数量
- 主照片占比
- 照片块分布
- 填充块分布
- 图片切片 / 颜色采样 / 纯色填充

### 导出与分享

- PNG
- JPEG
- Web Share API
- 下载
- 复制分享文案
