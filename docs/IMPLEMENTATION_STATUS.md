# POiS 当前实现进展

## 1. 当前阶段判断

当前项目处于 `单主图编辑器收敛期`。

这一轮不是继续扩展拼图模板，而是把产品模型缩到更稳定的一条主路径：

- 首页上传 `主照片`
- 编辑页生成 `双 panel 画板`
- 主照片负责真实内容
- 另一侧是 `填充块`
- 填充块可以继续使用底板样式，也可以额外上传一张 `填充块照片`

当前更准确的定位是：

- `核心体验已可运行`
- `单主图 + 可选填充块图` 已落地
- `波点交换渲染` 已跑通
- `接下来重点是交互细节、视觉一致性和测试补足`

## 2. 当前用户流程

### 首页

- 当前直接进入编辑器空状态
- 上传入口在画布居中区域
- 首页旧引导页与额外说明文案已移除

### 编辑页

- 顶部旧图片栏已经删除
- 桌面端是 `左侧主画板 + 右侧编辑面板`
- 移动端沿用单画板 + 折叠面板结构
- 右侧面板固定为：
  - `布局`
  - `填充块`
  - `波点`
- 右上角只保留 `导出 PNG`

### 填充块照片

- 不再从首页上传第二张图
- 需要替换填充块时，在 `填充块` 面板里手动上传
- 当前填充块照片只负责渲染，不提供单独编辑 UI

## 3. 当前画板模型

### 固定双 panel

画板固定由两个块组成，只支持两种方向：

- `左右分块`
- `上下分块`

### 单主图默认布局

当项目只有一张主照片时：

- 先根据主照片原始宽高比计算真实照片块
- 再生成一个 `同尺寸` 的填充块
- 两块直接拼接，不再额外包两个独立等大外框
- 默认缩放到视口中央，约占可见区域 `50%`

### 填充块渲染规则

- 若 `fillPhotoId` 存在：
  - 次 panel 按照片渲染
- 若 `fillPhotoId` 不存在：
  - 次 panel 按底板样式渲染

## 4. 当前状态与数据流

### 核心状态

当前核心状态已经收敛为：

- `photoIds`
- `fillPhotoId`
- `photoCrops`
- `panelDirection`
- `base`
- `dots`
- `exportFormat`

其中：

- `photoIds` 当前只承担主照片
- `fillPhotoId` 单独承担填充块照片
- `activeSources` 由主照片和填充块照片合并得到

### 裁切规则

- 主照片裁切仍然使用 `x / y / scale`
- 当前主照片默认用 `contain` 初始 fit 进入画板
- 只有 `primary panel` 提供裁切交互
- 填充块照片当前按默认渲染，不暴露独立裁切面板

## 5. 当前波点系统

### 形状

当前 UI 只保留：

- `五角星`
- `水滴`
- `雪花`
- `圆形`
- `文本`
- `爱心`
- `花朵`
- `Kitty`

其中：

- `水滴` 路径已重新调整
- `雪花` 路径已重新调整
- `爱心 / 花朵 / Kitty` 使用固定 silhouette 路径

### 分布规则

当前 `distribution` 已改成三种：

- `random`
- `single-side`
- `double-side`

对应行为如下：

- `random`
  - primary / secondary 各自独立随机点位
- `single-side`
  - 只在 primary 侧绘制波点
- `double-side`
  - 两侧共用同一组点位

同时支持画笔方式：

- `same-size`
- `large-to-small`
- `small-to-large`

### 采样关系

当前波点渲染是相互取样：

- primary panel 的波点从 secondary surface 取样
- secondary panel 的波点从 primary surface 取样

这使得两边的颜色 / 图片内容会发生交换式拼贴，更接近参考站效果。

## 6. 当前面板配置

### 布局面板

当前只保留：

- 标题 `布局`
- 两个切换按钮：
  - `左右分块`
  - `上下分块`

以下旧项已从当前主交互中删除：

- 说明文案
- 画幅
- 主 panel 占比
- 留白
- 块间距

### 填充块面板

当前保留：

- `上传填充块照片`
- `底板样式`
- `主色`
- `辅色`
- `条纹粗细`

当前已删除：

- `主题风格`
- `双色底`
- `副 panel 显示波点`

### 波点面板

当前保留：

- `形状`
- `点位分布`
- `画笔方式`
- `撤回`
- `清空`
- `点大小`
- `透明度`
- `大小变化`
- `文本内容`
- `文字大小`

当前已删除：

- 顶部说明文案
- 形状下方说明文案
- `采样方式`
- `大小差异`
- `总点数`
- `装饰点`
- `随机一下`

## 7. 当前已完成能力

- 首页上传主照片并进入编辑页
- 编辑页上传填充块照片
- 画板默认 fit 到视口中心
- 拖动画布平移
- 滚轮缩放 / 双指缩放
- 双击重置默认视口
- 主照片拖动与缩放裁切
- 波点跨 panel 交换渲染
- 手动画点撤回 / 清空
- PNG 直接导出下载
- 刷新后不恢复旧编辑会话

## 8. 当前仍待完善

- 填充块照片的独立编辑能力
- 更多针对参考站的视觉一致性校准
- 自动化测试与冒烟测试补齐
- redo
- 更细的移动端手势手感调优

## 9. 当前结论

一句话总结：

`POiS 目前已经从“泛化拼图原型”收敛成“单主图 + 可选填充块图”的双 panel 编辑器，当前最重要的是把这条路径继续磨准，而不是重新扩散功能面。`

部署前建议至少完成以下动作：

- 运行 `npm test`
- 运行 `npm run build`
- 用一张横图和一张竖图分别验证预览与导出
- 验证花朵、Kitty、文本、爱心四类特殊形状
- 验证单侧波点和双侧波点的手动画点逻辑
